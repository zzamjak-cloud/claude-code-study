매우 흥미롭고 도전적인 프로젝트입니다! 구상하신 **"연간 타임라인(Timeline) 형태의 리소스 스케줄러"**는 UI/UX 복잡도가 상당히 높은 기능입니다. Claude Code와 함께 개발하신다면, **기능을 아주 작게 쪼개서 단계별로 프롬프팅**하는 것이 성공의 핵심입니다.

전체적인 청사진을 **4단계 로드맵**으로 정리해 드립니다.

---

### 1단계: 데이터 모델링 (Firebase Firestore)

가장 먼저 데이터 구조가 튼튼해야 합니다. UI를 그리기 전에 데이터가 어떻게 저장될지 정의하세요.

* **Users Collection:** 팀원 정보 (이름, 프로필 사진, 직무 등)
* **Schedules Collection:** 핵심 데이터입니다.
```json
{
  "id": "schedule_123",
  "userId": "user_abc", // 누구의 일정인지
  "title": "프로젝트 A 기획",
  "startDate": "2026-01-15T09:00:00", // ISO String or Timestamp
  "endDate": "2026-01-20T18:00:00",
  "color": "#FF5733", // 카드 색상
  "type": "work" // 업무, 휴가 등 타입 구분
}

```



> **Tip:** Firebase의 `onSnapshot`을 사용하면 별도의 새로고침 구현 없이 팀원이 일정을 옮기는 즉시 다른 사람 화면에도 실시간으로 반영됩니다.

---

### 2단계: 핵심 UI 구조 - "무한 스크롤이 아닌 가상 스크롤"

가장 중요한 기술적 결정 사항입니다. 1년(365일) × 팀원 N명의 그리드를 전부 DOM에 렌더링하면 브라우저가 매우 느려집니다.
따라서 각 개인의 일정을 별도 탭으로 두고, 통합탭을 만들어서 모든 팀원의 일정을 모아서 렌더링하는 방식을 취합니다.

* **Virtualization (가상화) 필수:** 화면에 보이는 기간(예: 1월 1일~1월 15일)만 렌더링하고, 스크롤할 때마다 DOM을 갈아끼우는 방식을 써야 합니다.
* 추천 라이브러리: `react-window` 또는 `react-virtualized`


* **CSS Grid 활용:**
* 가로축: 날짜 (1일 = 예: `50px` 고정)
* 세로축: 팀원 (Row)


* **날짜 계산 로직:** `date-fns` 라이브러리를 사용하여 날짜 간의 차이를 픽셀로 변환하는 유틸리티 함수가 필요합니다.

---

### 3단계: 인터랙션 구현 (Drag, Resize, Snap)

이 부분이 사용자 경험(UX)의 핵심입니다.

1. **라이브러리 선정:**
* 처음부터 다 짜기보다는 **`dnd-kit`** (최신, 가벼움) 혹은 **`react-rnd`** (Resizable & Draggable 특화) 사용을 추천합니다. 특히 `react-rnd`는 크기 조절과 이동이 동시에 필요할 때 유용합니다.


2. **Snap to Grid (자석 효과) 로직:**
* 그리드 1칸의 너비가 `50px`이라고 가정할 때:
* 드래그 종료 시 좌표 `x`를 `Math.round(x / 50) * 50`으로 계산하여 강제로 위치를 보정합니다. 이렇게 하면 카드가 항상 날짜 칸에 딱 맞게 떨어집니다.


3. **날짜 역산:**
* 카드가 놓인 `x` 좌표 / `50px` = `시작일로부터 며칠 뒤`인지 계산하여 DB에 업데이트합니다.

### 4단계: Claude Code와의 협업 전략 (프롬프트 가이드)

한 번에 "이거 만들어줘"라고 하면 AI도 실수합니다. 다음과 같은 순서로 요청해 보세요.

1. **Skeleton 구축:** "React, Vite, Tailwind CSS로 프로젝트를 세팅해줘. 팀원 7명이 있고 우선은 팀원 1명의 일정을 등록할 수 있는 가로로 1월 1일부터 12월 31일까지 스크롤되는 빈 그리드(Grid)를 먼저 만들어보자."
2. **Card Component:** "그리드 위에 절대 위치(absolute)로 배치될 'ScheduleCard' 컴포넌트를 만들어줘. `react-rnd`를 사용해서 가로로 사이즈 조절과 이동이 가능하게 해줘."
3. **Snap Logic:** "카드를 드래그하다 놓았을 때, 50px 단위로 위치가 딱 맞게(Snap) 조정되는 함수를 추가해줘."
4. **Firebase 연동:** "이제 이 카드의 위치 정보를 바탕으로 시작 날짜와 종료 날짜를 계산해서 Firebase Firestore에 저장하고 불러오는 로직을 연결해줘."

# 핵심 기능 나열

### 팀원 관리 기능 (관리자 기능으로 제한)
1. 팀원 추가 기능 : 신규 팀원이 생겼을때 팀원을 추가하면 탭이 늘어나도록 처리.
2. 팀원 삭제 기능 : 잘못 등록한 팀원을 삭제하는 기능, 삭제시 안전성을 위해서 "OOO를 삭제" 라고 텍스트를 입력후 확인을 눌러야 삭제되도록 처리
3. 팀원 숨김 기능 : 팀원이었다가 타팀으로 전배시 숨김 처리하면 탭은 사라지고 보관함 탭에서 다시 꺼내올 수 있는 구조.
4. 팀원 탭 드래그 기능 : 탭을 드래그해서 순서를 변경할 수 있습니다. (통합 탭은 제일 첫번째 순서로 고정됩니다.)

### 일정 등록 화면
1. 연간 일정 그리드에서 특정 셀을 Ctrl + 드래그해서 끝나는 셀까지의 크기로 카드를 생성합니다. 만약 드래그 중에 다른 일정카드와 겹칠 경우에는 겹친 부분이 빨간색으로 표시됩니다. 직접 드래그해서 겹치지 않도록 사용자가 이동해야 합니다. 생성된 카드를 더블 클릭해서 텍스트 입력 및 수정합니다. 카드 위에서 마우스 우클릭시, 카드 배경 컬러 및 텍스트 컬러 설정 프리셋과 함께 링크, 삭제 버튼이 호출됩니다. 링크 버튼 클릭시 링크 정보를 입력할 수 있습니다. 링크 정보가 등록될 경우, 카드에 링크 바로가기 아이콘이 표시됩니다. 삭제 버튼 클릭시 "일정을 삭제하시겠습니까?"라는 알림창이 뜨고, "삭제" 버튼을 클릭하면 삭제됩니다.
2. 팀원 개인별로 고유한 일정탭을 가지게 되며, 통합탭이 따로 존재해서 개인별 등록된 일정을 모아서 출력합니다. 일정의 행 세션의 순서는 탭 순서입니다.
3. 전역 기능이 필요합니다. 
    1. 필터링 기능 : 지정된 특정 기간만 표시할 수 있습니다.
    2. 특이사항 : 공휴일이나, 사내 평가 기간, 사내 행사, 서비스 오픈 등을 등록하면 각 팀원 탭에 공통으로 적용됩니다.
